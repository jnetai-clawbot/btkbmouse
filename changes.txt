# BT KB & Mouse - Change Log

## v1.0.3

### Critical HID Fix
- Fixed "Waiting for host" issue: now calls `BluetoothHidDevice.connect(device)` after `registerApp()`
  to actively establish the HID channel with the Linux host
- Without `connect()`, the Android device advertised HID but never initiated the connection,
  so the Linux host never saw it as a usable HID keyboard/mouse
- `retryHidRegistration()` now re-resolves the target device from the stored address
- Connection flow: pair → connect (BT) → registerApp → connect (HID) → send reports

## v1.0.2

### HID Registration Fix
- Fixed "HID Not Registered" issue when connecting to Linux (ARM 64-bit Ubuntu) hosts
- App now correctly tracks HID registration state in `registerApp()` callback
- `isHidAppRegistered` set immediately on successful `registerApp()`, before host connects
- Added `isHidConnected` field to track actual host connection state separately
- Connection status now shows: "HID Connected", "HID Registered (Waiting for host...)", or "HID Not Registered"
- Added automatic retry logic: up to 5 attempts with backoff (0s, 2s, 4s, 8s, 16s)
- HID callback `onConnectionStateChanged` triggers re-registration on connect events
- Added "Retry HID Registration" button in Connection screen
- HID proxy disconnection also triggers retry for robust recovery

## v1.0.1

### HID Implementation
- Full Bluetooth HID device implementation with proper HID report descriptors
- Keyboard: Full key mapping with shift support (a-z, 0-9, symbols, enter, space, tab, backspace)
- Mouse: Precise relative movement reports with 8-bit X/Y/wheel encoding
- Left click, right click, scroll wheel support
- HID proxy registration via BluetoothProfile.ServiceListener
- Combined keyboard + mouse descriptor registered as subclass combo device
- QOS settings configured for best-effort HID transport
- Send text as sequence of key presses with proper delays
- Mouse reports include proper button bitmask encoding

### Split View Screen
- Combined touchpad + keyboard on single screen
- Instant mode: Text field sends keystrokes as typed
- Clipboard mode: Recent sent history for quick replay
- Presets mode: Saved phrases one-tap send
- Touchpad with drag (move), tap (left click), long-press (right click)
- Sensitivity slider shared between touchpad and keyboard
- Scroll buttons for wheel scrolling
- Toggle keyboard panel visibility with icon

### Touchpad Improvements
- Left click sends proper HID button 1 report
- Right click sends proper HID button 2 report
- Long press gesture for right-click

### Bug Fixes
- Fixed Flow.first() compilation errors in ViewModel
- Fixed missing pointerInput import in Components.kt
